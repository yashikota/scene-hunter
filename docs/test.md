# test

テストについては単体テストを基本的に全ての処理について書くが、以下の方針に従う  

- モックは原則用いない。DBやKVS、Blobも `testcontainers` を用いることでモックを使わない単体テストを行う
  - ただし外部APIを叩く処理などは例外的にmockを使用して良い
  - その際は `gomock` を使用する
- private関数はテストを書かない
  - public関数経由でテストができるため
- テストはテーブル駆動テストとして書く
  - テストケースはmapで書く
  - テストケースを1列にする
  - フィールド名はいらない
  - ヘルパー関数が使えそうなら積極的に使う
- テストで扱う時刻は全てUTCとする
